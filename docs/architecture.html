GCS Bucket Module

Executive Summary

A production-ready Terraform module for provisioning Google Cloud Storage buckets with comprehensive data management capabilities. Supports four storage classes (STANDARD, NEARLINE, COLDLINE, ARCHIVE) with Autoclass for automatic transitions, lifecycle rules with all condition types, object versioning, retention policies with irreversible locking, CORS configuration, CMEK encryption via Cloud KMS, uniform bucket-level access with public access prevention, Pub/Sub event notifications, access logging, static website hosting, custom dual-region placement, soft delete policies, and granular IAM bindings.

Component Breakdown

google_storage_bucket.this

Core bucket resource with storage class, location (region/dual-region/multi-region), uniform bucket-level access, public access prevention (enforced/inherited), requester pays, event-based holds, versioning, and Autoclass for automatic storage class transitions.

Lifecycle Rules

Comprehensive lifecycle management with Delete, SetStorageClass, and AbortIncompleteMultipartUpload actions. Conditions include age, created_before, with_state, matches_storage_class/prefix/suffix, num_newer_versions, days_since_noncurrent_time, and custom time conditions.

google_storage_bucket_iam_member.bindings

Flattened IAM bindings for role-based access (objectViewer, objectCreator, objectAdmin, etc.) using for_each over role-member pairs.

google_storage_notification.notifications

Pub/Sub notifications for object events (FINALIZE, METADATA_UPDATE, DELETE, ARCHIVE) with object name prefix filtering and custom attributes.

Data Flow

  • Upload: Objects are written to the bucket via gsutil, client libraries, or Transfer Service. CMEK encryption is applied at rest.
  • Lifecycle: Objects transition through storage classes (STANDARD to NEARLINE to COLDLINE to ARCHIVE) based on age or access patterns (Autoclass).
  • Versioning: When enabled, each object modification creates a new version. Noncurrent versions are managed by lifecycle rules.
  • Notifications: Object events trigger Pub/Sub messages for downstream processing (Cloud Functions, Cloud Run, Dataflow).
  • Retention: Objects are locked for the retention period. Locked policies are irreversible.
  • Soft Delete: Deleted objects are recoverable within the soft delete retention window.

Industry Adaptation

  • Financial Services: Locked retention policies for regulatory compliance, CMEK encryption, uniform access, audit logging, soft delete for data recovery.
  • Healthcare: HIPAA-compliant configuration with public access prevention, CMEK, access logging, lifecycle rules for PHI data retention.
  • Media / CDN: Static website hosting with CORS, multi-region buckets for low-latency global delivery, Pub/Sub notifications for transcoding pipelines.
  • Data Lake: Autoclass for cost optimization, lifecycle rules for tiered archival, versioning for data lineage, Pub/Sub for event-driven ETL.

Production Readiness Checklist

  • Enable uniform_bucket_level_access = true (default)
  • Set public_access_prevention = "enforced" (default)
  • Enable CMEK encryption with a dedicated Cloud KMS key
  • Configure lifecycle rules for storage class transitions and object deletion
  • Enable versioning for data protection
  • Set force_destroy = false (default) to prevent accidental data loss
  • Configure access logging to a separate bucket
  • Set retention policy for compliance-critical data
  • Enable soft delete with appropriate retention duration
  • Configure Pub/Sub notifications for event-driven workflows
  • Assign least-privilege IAM bindings
  • Apply labels for cost tracking and resource organization

Deployment

module "bucket" {
  source = "github.com/kogunlowo123/terraform-gcp-gcs-bucket"

  project_id         = "my-project"
  name               = "my-project-data-lake"
  location           = "us-central1"
  storage_class      = "STANDARD"
  versioning_enabled = true

  encryption = {
    default_kms_key_name = "projects/my-project/locations/us-central1/keyRings/ring/cryptoKeys/key"
  }

  lifecycle_rules = [
    {
      action    = { type = "SetStorageClass", storage_class = "NEARLINE" }
      condition = { age = 30 }
    },
    {
      action    = { type = "SetStorageClass", storage_class = "COLDLINE" }
      condition = { age = 90 }
    },
    {
      action    = { type = "Delete" }
      condition = { age = 365 }
    }
  ]

  soft_delete_policy = { retention_duration_seconds = 604800 }

  notifications = [
    {
      topic        = "projects/my-project/topics/gcs-events"
      event_types  = ["OBJECT_FINALIZE"]
    }
  ]

  iam_bindings = {
    "roles/storage.objectViewer" = ["serviceAccount:reader@my-project.iam.gserviceaccount.com"]
    "roles/storage.objectAdmin"  = ["serviceAccount:writer@my-project.iam.gserviceaccount.com"]
  }

  labels = { team = "data", env = "production" }
}
